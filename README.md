Контейнеризация на C++ Уеб Приложение с Docker Compose

Този проект е създаден като практическо задание на тема Контейнеризация на проект с Docker Compose. Той представлява уеб сървър, разработен на C++, който комуникира с Redis база данни за броене на потребителските посещения (Hit Counter) в реално време.

Как работи всеки от компонентите

    Бекенд Услуга (backend)

    Технологии: C++11, cpp-httplib (HTTP Сървър), hiredis (C клиент за Redis).

    Роля: Слуша на порт 8080. При всяко отваряне на уеб страницата, изпраща команда INCR hits към Redis, взима новата стойност и я връща като динамичен HTML код към браузъра на потребителя.

    База Данни (db)

    Технологии: Redis:7-alpine (Официален олекотен образ).

    Роля: Съхранява броя на посещенията в оперативната памет за максимална бързина при работа.

Комуникация между услугите

Комуникацията между двата контейнера е реализирана по следния начин:

    Изолирана мрежа: Двата контейнера комуникират в споделена виртуална мрежа от тип bridge (app-network), дефинирана в compose.yml.

    Service Discovery: Вместо твърди IP адреси, C++ приложението намира базата данни директно чрез нейното име в мрежата — db, благодарение на вградения DNS в Docker.

    Динамична среда: Хостът на базата данни се подава на приложението чрез променлива на средата REDIS_HOST=db, която се чете в C++ кода.

    Контрол на стартирането: Използва се директивата depends_on, за да се гарантира, че Redis стартира преди уеб сървъра.

Как се изгражда и стартира проектът

За да стартирате проекта локално, трябва да имате инсталиран Docker и Docker Compose.

Стъпки за стартиране:

    Клонирайте проекта и влезте в неговата папка:
    git clone https://github.com/22123-ship-it/cpp-redis-counter.git
    cd cpp-redis-counter

    Стартирайте оркестрацията с командата:
    sudo docker compose up --build

    Отворете уеб браузър и отидете на адрес:
    http://localhost:8080

При всяко презареждане на страницата броячът на посещенията ще се увеличава.

За спиране на контейнерите използвайте командата:
sudo docker compose down
